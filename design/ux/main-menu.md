# UX Spec: Main Menu

> **Status**: In Design
> **Author**: ux-designer
> **Last Updated**: 2026-04-18
> **Journey Phase(s)**: Session Start — 玩家首次接触游戏
> **Template**: UX Spec

---

## Purpose & Player Need

**玩家目标**：确认两位玩家准备就绪，一起进入游戏

**核心功能**：
- 让两位玩家看到"这就是游戏"的第一印象
- 传达"梦境战斗"的世界观基调
- 确认双人合作就位后，过渡到 Boss Intro

**如果这个屏幕不存在或难以使用**：
- 玩家不知道如何开始游戏
- 缺少"这就是游戏"的第一视觉印象
- 两人不知道需要同时按键

---

## Player Context on Arrival

**何时看到**：游戏启动后，程序加载完成

**前置活动**：无 — 这是游戏的第一个屏幕

**玩家情绪状态**：
- 期待、轻松 — 还没有压力
- 好奇 — 想看到游戏是什么样的
- 两人坐在一起，准备手柄/键盘

**情感基调**：温暖、欢迎的感觉。"梦境战斗"的氛围从主菜单就开始铺垫。

---

## Navigation Position

**根入口**：主菜单是游戏启动后的第一个屏幕，是整个游戏导航树的根节点。

```
[Root] Main Menu
    ├── Boss Intro → Gameplay HUD → Pause Menu
    └── (未来扩展) Settings
```

**特点**：
- 主菜单是起点，不是中途入口
- 从 Boss Intro 开始后，无法返回主菜单（单向流程）
- 未来扩展：可从 Pause 菜单进入 Settings，再返回游戏

## Entry & Exit Points

### 进入来源

| Entry Source | Trigger | Player carries this context |
|---|---|---|
| 游戏启动 | 程序加载完成 | 无 |
| (未来) 从 Settings 返回 | Settings → 返回 | 无 |

### 退出目标

| Exit Destination | Trigger | Notes |
|---|---|---|
| Boss Intro | 两位玩家各自按下任意键，P1+P2 同时就绪 | Main Menu → Boss Intro 是单向过渡，无法返回主菜单 |

### 双向就绪机制

两人必须**同时就绪**才能进入游戏：
- P1 按下任意键 → P1 图标高亮，显示 "P1 Ready"
- P2 按下任意键 → P2 图标高亮，显示 "P2 Ready"
- P1 + P2 同时就绪 → 触发过渡到 Boss Intro
- 如果 P1 或 P2 超过 10 秒无操作，就绪状态清除（防止玩家误操作）

## Layout Specification

### Information Hierarchy

**层级 1（最重要）**：
1. 游戏标题 — "今日Boss：打工吧！"
2. "按任意键开始" 提示

**层级 2（次要）**：
3. 双人合作提示 — "请两位玩家同时按键"
4. 两位玩家的就绪状态图标

### 布局方案：沉浸式（方案 B）

**视觉概念**：
- 全屏黑暗背景
- 中央暖色 spotlight 打光（模拟舞台聚光灯）
- 游戏标题在光圈中央偏上
- "按任意键开始" 在光圈中央偏下
- 底部小字提示双人合作
- 两位玩家就绪图标显示在屏幕底部两侧

**Spotlight 效果**：
- 颜色：暖橙色 #F5A623（与 P1 橙色呼应）
- Radial gradient，从上方打下来形成圆形光圈
- 微微的光晕向外扩散

## States & Variants

### Screen States

| State | Trigger | 视觉变化 |
|-------|---------|---------|
| Default | 初始状态 | Spotlight + 标题 + "按任意键开始" 呼吸脉冲 |
| P1Ready | P1 按下任意键 | P1ReadyIcon 橙色高亮 |
| P2Ready | P2 按下任意键 | P2ReadyIcon 蓝色高亮 |
| BothReady | P1+P2 同时就绪 | 两人图标同时高亮，PressStart 加速脉冲 |
| Transitioning | BothReady 后 0.5s | Spotlight 扩大，淡出到 Boss Intro |
| IdleReset | 任意玩家超过 10 秒无操作 | 就绪状态清除，图标恢复半透明 |

### Empty State / Error State

主菜单是纯视觉展示屏幕，没有数据依赖，因此没有 empty state。

如果程序加载失败（极罕见情况）：
- 显示简单错误提示："加载失败，请重启游戏"
- 提供"重试"按钮

### Loading State

程序加载期间：
- 显示简单 loading 提示或进度条
- 完成后直接淡入 Main Menu（不使用 loading screen）

## Interaction Map

**输入设备**：Keyboard/Mouse（主）+ Gamepad（部分支持）

主菜单的交互非常简单 — 只需要捕获按键状态。

### P1 交互

| 按键 | 动作 | 即时反馈 | 结果 |
|------|------|---------|------|
| P1 任意键（键盘/手柄） | P1 Ready | P1ReadyIcon 从半透明变为橙色高亮 | P1 标记为就绪 |

### P2 交互

| 按键 | 动作 | 即时反馈 | 结果 |
|------|------|---------|------|
| P2 任意键（键盘/手柄） | P2 Ready | P2ReadyIcon 从半透明变为蓝色高亮 | P2 标记为就绪 |

### 双方同时就绪

| 条件 | 动作 | 即时反馈 | 结果 |
|------|------|---------|------|
| P1 + P2 同时就绪 | 触发进入 | Spotlight 扩大，淡出 | 过渡到 Boss Intro |

### 注意事项

- **任意键**：不需要特定按键，任何键都可触发就绪
- **去抖动**：P1/P2 就绪后，10 秒无操作则自动清除就绪状态（防止误触）
- **同时就绪检测**：检测 P1 和 P2 的按键press（frame-accurate），不检测 hold

## Events Fired

| 玩家动作 | 触发事件 | 携带数据 |
|---------|---------|---------|
| P1 按下任意键 | `player_ready` 或 `p1_ready` | player_id = 1 |
| P2 按下任意键 | `player_ready` 或 `p2_ready` | player_id = 2 |
| P1 + P2 同时就绪 | `both_players_ready` | — |
| 进入 Boss Intro | `start_game` 或 `transition_to_boss_intro` | — |

**注意**：这些是 UI 层的 input 事件，不是全局游戏事件（游戏事件由 GameState 管理）。

## Transitions & Animations

### Screen Enter Transition

**从程序加载 → Main Menu**：
- Fade in from black（0.5s）
- Spotlight 从小到大展开（0.3s）

### Screen Exit Transition

**从 Main Menu → Boss Intro**：
- Spotlight 扩大，淡出（0.5s）
- 同时 Boss Intro 淡入

### Press Start 呼吸脉冲动画

**规格**：
- 周期：0.8s（一次呼吸）
- 效果：透明度在 0.5 ~ 1.0 之间循环
- 实现：使用 `sin(time * PI)` 或 Godot Tween

### 就绪状态切换动画

**P1/P2 图标切换**：
- 未就绪 → 就绪：Tween opacity 0.3 → 1.0（0.2s）+ 轻微缩放弹跳

### BothReady 触发动画

**两人同时就绪后**：
- PressStart 脉冲加速（从 0.8s 变为 0.3s）
- 持续 0.5s 后触发 exit transition

### Idle Reset

**10 秒无操作就绪清除**：
- 图标高亮 → 半透明：Tween opacity 1.0 → 0.3（0.3s）

### Reduced Motion

如果玩家启用了 reduced motion 选项：
- 呼吸脉冲动画改为静态（无透明度变化）
- 过渡动画改为 instant cut

## Data Requirements

主菜单是纯视觉展示屏幕，**不读取也不写入游戏状态数据**。

| 数据 | 来源系统 | 读/写 | 备注 |
|------|---------|--------|------|
| 无 | — | — | 主菜单不依赖任何游戏数据 |

**状态管理**：
- P1/P2 就绪状态由 MainMenuScreen 内部管理（UIState）
- 两位玩家就绪后，发出 `both_players_ready` 信号
- GameState 接收信号，执行 Boss Intro 过渡

## Accessibility

### Basic Tier 覆盖

| 功能 | 状态 | 实现说明 |
|------|------|---------|
| 最低字体大小 | **必须实现** | 标题 32px+，提示文字 20px+，底部小字 14px |
| 文本对比度 | **必须实现** | 所有文字/背景对比度 ≥ 4.5:1 |
| Reduced motion 选项 | **必须实现** | 呼吸脉冲动画可关闭，改为静态显示 |
| 键盘/手柄支持 | **必须实现** | 任意键都可触发，P1/P2 各支持键盘+手柄 |

### 颜色使用

**Spotlight**：#F5A623（暖橙色）— 用于 spotlight 光效
**文字**：白色 #FFFFFF — 高对比度

**注意**：主菜单不涉及 P1/P2 颜色区分（P1=橙色/P2=蓝色只在 HUD 中使用）。

### 屏幕阅读器支持

主菜单是纯展示屏幕，没有可交互的菜单项，屏幕阅读器支持有限。考虑到：
- 主菜单是游戏入口，不是导航菜单
- 主要交互（按键开始）没有可访问的替代方式
- Basic tier 不要求屏幕阅读器支持（仅要求菜单屏幕）

**建议**：如果未来需要屏幕阅读器支持，可以添加"按回车开始"作为键盘专用入口。

### 键盘/手柄导航

主菜单不需要 Tab 导航或 focus order — 任意键触发，没有焦点概念。

**例外**：如果未来添加了可访问选项（如"无障碍选项"入口），需要 Tab navigation。

## Localization Considerations

### 文本元素

| 元素 | 当前文字 | 字符数 | 本地化风险 |
|------|---------|--------|-----------|
| GameTitle | "今日Boss：打工吧！" | 9 | 低 — 中文原版 |
| PressStartPrompt | "按任意键开始" | 6 | 低 |
| CoopHint | "请两位玩家同时按键" | 10 | 低 |
| P1ReadyText | "P1 Ready" | 7 | 低 |
| P2ReadyText | "P2 Ready" | 7 | 低 |

### 布局风险

**GameTitle**："今日Boss：打工吧！" 是中文，在 1080p 屏幕上的spotlight 光圈内应能正常显示。

**风险项**：无 — 所有文字都是短句，不会超过布局边界。

**未来扩展**：如果添加其他语言的本地化（如英文版），注意：
- "按任意键开始" → "Press any key to start"（英文字符更长）
- 需确认 spotight 宽度能容纳最长翻译

## Acceptance Criteria

- [ ] 主菜单在程序加载完成后 1 秒内显示
- [ ] "按任意键开始" 文字有呼吸脉冲动画（0.8s 周期）
- [ ] P1 按下任意键后，P1ReadyIcon 从半透明变为橙色高亮（0.2s 内完成）
- [ ] P2 按下任意键后，P2ReadyIcon 从半透明变为蓝色高亮（0.2s 内完成）
- [ ] P1 + P2 同时就绪后，0.5s 内触发过渡动画
- [ ] 过渡动画持续 0.5s，Main Menu 淡出，Boss Intro 淡入
- [ ] P1/P2 就绪后 10 秒无操作，自动清除就绪状态
- [ ] 所有文字对比度 ≥ 4.5:1（白色文字在暗色背景上）
- [ ] 标题字号 ≥ 32px，提示文字 ≥ 20px，底部小字 ≤ 14px
- [ ] Reduced motion 选项启用时，动画改为 instant（无呼吸脉冲）
